EasyLanguage Reserved Words & Functions

Account Orders & Positions in EasyLanguage

There are a number of reserved words that allow you to access position and brokerage account information so you can more accurately tie strategies to your real-world trading. For example, you can now adjust the share/contract size of a strategy automated order to avoid exceeding buying power or margin limits. You can also check a real-world position for the purpose of keeping your strategy in sync with the real-world, or, check on the status of both positions in a  ”pairs” strategy.

In addition to the selection of Account Orders & Positions words listed below you can find others in the EasyLanguage Dictionary (in the TradeStation Development Environment) under the category: ”TradeManager”

Historical Testing:

Since most of these functions require a potentially slow out-of-process call, they will always return zero when one of the functions is called historically.

EasyLanguage Example:

Following is sample code to allow a limit entry order that has been unfilled in the real-world but filled in the strategy to be replaced.  The strategy will generate a market exit order when the above out-of-sync condition is detected.  This exit order will take the strategy flat and cancel the unexecuted entry order allowing a new entry order to be placed at a different limit price.

{ Long Exit Sync Strategy }

Vars:  MyRealPosition(0) ;

if LastBarOnChart = TRUE and MarketPosition = 1 then Begin

MyRealPosition = GetPositionQuantity(GetSymbolName, GetAccountID) ;

if MyRealPosition = 0 then Begin

Sell("SYNCLX") next bar at market ;

End ;

End ;

 

{ Short Exit Sync Strategy }

Vars:  MyRealPosition(0) ;

if LastBarOnChart = TRUE and MarketPosition = -1 then Begin

MyRealPosition = GetPositionQuantity(GetSymbolName, GetAccountID) ;

if MyRealPosition = 0 then Begin

Buy to Cover("SYNCSX") next bar at market ;

End ;

End ;
